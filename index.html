Working with GitFlow

About

GitFlow is an extension of Git for team developer projects, where many branches can be created, deleted, and merged in a dynamic way. GitFlow provides security for the main/master branch, ensuring that the code sent to production is reliable.

Branches

  • Master/Main: The main branch contains all production code, and new features are merged into it through either a Hotfix or a new Release.
  • Develop: This branch holds code for the upcoming deployment and includes unreleased features that will eventually be merged into the Master branch.
  • Feature: Feature branches are used for specific feature development, following a naming convention like 'feature/alura-forum.' They are always created from the Develop branch and are removed after merging. They should not interact with the master branch, only with the develop branch.
  • Hotfix: A Hotfix branch is created from the master to address immediate production issues and is merged into both Master and Develop when completed. This differs from Feature Branches, which start from Develop. Additionally, closing a Hotfix Branch involves creating a version tag for the project.
  • Release: After completing a development stage, all features and Hotfixes are merged into the Develop branch. To include these in the Master branch, a Release branch is created, acting as a staging environment. Once merge tests are successful, the Release branch is deleted. If bugs are found, it's synchronized with Develop. Creating a tag with the new software version is essential to maintain a complete development history.

GitFlow Basic Commands

git flow initInitialization to start using GitFlow in your project, you need to initialize it.
git flow feature start FEATURE_NAMEStarting a new feature. Create a branch (based on develop) and it will automatically check out to this branch.
git flow feature finish FEATURE_NAMEThis command will perform the following actions: Merge with the develop branch; Delete the feature branch; Check out to the develop branch. To finish a feature, you should execute this command inside the feature branch you wish to finalize.
git branchTo check all active branches in the project.
git flow feature publish FEATURE_NAMEPublishing a feature. Other developers besides you can continue working on this feature.
git flow release start MY_VERSIONStarting a new version. Assists in preparing a new production version.
git flow release publish MY_VERSIONPublishing a version. It's wise to publish the version branch after creating it to allow commits by other developers.
git flow release finish MY_VERSIONFinishing a version. When finishing a version, it performs several actions: Merges the MY_VERSION branch into MASTER. Merges the MY_VERSION branch and checks out to DEVELOP. Tags the version with its name. Removes the MY_VERSION branch.
git flow hotfix start MY_HOTFIXCreates a branch based on MASTER. The argument MY_HOTFIX, in this case, marks the defective version in production.
git flow hotfix finish MY_HOTFIXIt will perform a merge into MASTER and DEVELOP. The merge into MASTER will be tagged.
git reset --hardIf you encounter an error like 'fatal: Working tree contains unstaged changes. Aborting', you can try this command to resolve it when working in certain GitFlow branches.
